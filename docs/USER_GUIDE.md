# 智慧分身用户使用指南

> 智慧分身（Memory OS）是一个隐私优先的个人记忆与对话系统。本指南帮助你快速上手并了解所有核心功能。

## 1. 快速开始

### 1.1 注册账户

1. 打开智慧分身 Web 控制台（默认地址 `http://localhost:8000`）。
2. 进入注册页面，填写邮箱和密码。
3. 提交后系统会使用 bcrypt 哈希算法加密你的密码并创建账户。
4. 注册成功后即可使用邮箱和密码登录。

> **提示**：密码以不可逆的哈希形式存储在本地数据库中，系统无法看到你的明文密码，请妥善保管。

### 1.2 登录系统

1. 在登录页面输入注册时的邮箱和密码。
2. 登录成功后，系统会签发 JWT 认证凭证，后续所有 API 请求均需携带该凭证。
3. 登录后默认进入 **Chat 中枢** 页面，你可以从这里开始使用所有功能。

> **提示**：如果你在“设置”中开启了 API Key 鉴权，还可以在左侧边栏底部的 API Key 输入框填入接口密钥，用于保护本机接口。

## 2. 核心功能

### 2.1 Chat 中枢

Chat 是系统的主入口。你不需要先选择“故事”或“报告”，直接用自然语言表达意图即可。

**使用步骤**：

1. 在左侧导航点击 **Chat** 进入对话页面。
2. 在“消息”输入框中输入你的问题，例如：
   - “我最近是不是太固执了？”
   - “上次让我真正放松是什么时候？”
   - “最近哪些事情让我反复感到消耗？”
3. （可选）在“指定 Skill”下拉框中选择一个 Skill，或保持“自动判断”让系统根据意图自动路由。
4. （可选）在“权限 Token”输入框填入已创建的授权 Token。
5. 勾选“本次确认授权读取脱敏记忆”表示你同意本次调用读取你的脱敏记忆。
6. 点击“发送”。

**处理流程**：

- 系统判断你的意图（回忆、反思、创作、统计等）。
- 本地向量库检索相关记忆卡片。
- 组装脱敏上下文，按需调用 Skill 或 MCP 工具。
- 返回结果时标注引用了哪些记忆卡片，区分事实、推断和建议。

### 2.2 录音采集

将一段语音录音自动转写并提炼为脱敏记忆卡片。

**使用步骤**：

1. 在左侧导航点击 **录音采集**。
2. 点击“开始录音”按钮，授权浏览器使用麦克风。
3. 录制完成后点击“停止”，可在播放器中预览。
4. 点击“上传并转写”，系统会：
   - 将原始音频保存在本地 `data/recordings/` 目录。
   - 使用本地 `faster-whisper` 模型完成转写（无需联网）。
   - 自动将转写文本提炼为脱敏记忆卡片并写入记忆仓库。
5. 你也可以通过“选择已有音频文件上传”直接上传本地音频文件。

> **隐私提示**：原始音频仅本地存储，不上传云端。转写在本地完成。你可以在配置中设置 `auto_delete_after_days` 自动清理过期录音。

### 2.3 记忆仓库

手动新增、查看和删除记忆卡片。

**新增记忆**：

1. 在左侧导航点击 **记忆仓库**。
2. 在“新增记忆”表单中填写：
   - **事件摘要**：脱敏后的事件描述。
   - **时间范围**：如“上午”“下午”“今天”。
   - **情绪**：如“紧张但坚定”。
   - **洞察**：从事件中获得的感悟。
   - **标签**：如“工作, 沟通”。
   - **性格侧面**：如“执着, 责任感”。
3. 点击“写入记忆”。

**查看与删除**：

- 右侧“记忆仓库”列表按最新创建时间展示所有记忆卡片。
- 每条记忆支持查看详情和删除操作。

> **提示**：手动写入的记忆也应保持脱敏，不要直接填入真实人名、电话或精确地址。

### 2.4 今日故事

基于今日脱敏记忆生成融合真实与虚构的叙事。

**使用步骤**：

1. 在左侧导航点击 **今日故事**。
2. （可选）在“故事方向提示”中输入你希望故事的方向，例如“从今天一次坚持自我表达的瞬间出发”。
3. 勾选“本次确认授权读取脱敏记忆用于生成故事”。
4. 点击“生成今日故事”。

**输出结构**：

- **故事正文**：魔幻现实主义风格的个人叙事。
- **今日真实锚点**：标注故事中哪些事件来自真实记忆。
- **文学虚构元素**：标注哪些设定属于文学想象。
- **今日性格侧面**：当天最突出的性格特质。
- **续写钩子**：结尾留下的选择或悬念，方便你续写。

### 2.5 Skill Registry

查看系统已注册的场景能力（Skill）。

**使用步骤**：

1. 在左侧导航点击 **Skill Registry**。
2. 页面展示所有已注册的 Skill，包括名称、用途、触发词、所需权限和输出结构。
3. 内置示例 Skill 包括：
   - **daily_story**：每日故事生成。
   - **weekly_review**：周复盘。
   - **self_awareness**：自我觉察。

> **提示**：Skill 通过配置文件接入，不写进核心代码。你可以在 `skills/` 目录下新增 Skill。

### 2.6 MCP 工具

查看系统已注册的 MCP 工具。

**使用步骤**：

1. 在左侧导航点击 **MCP 工具**。
2. 页面展示所有已注册的工具 manifest，包括名称、用途、启用状态和所需权限。
3. 工具默认关闭，启用前需要配置权限、输入预览和审计策略。

> **提示**：你可以在 `tools/` 目录下新增工具配置。工具调用前会进行权限校验，调用后记录审计日志。

### 2.7 授权与凭证

创建授权 Token，生成本地哈希凭证。

**创建授权**：

1. 在左侧导航点击 **授权与凭证**。
2. 在“创建授权”表单中填写：
   - **授权对象**：如 `daily_story`。
   - **权限范围**：如 `memory:read:desensitized, story:write`。
3. 点击“创建授权”，系统返回授权 Token。
4. 该 Token 可用于 Chat、Skill 或工具调用，会自动填入 Chat 的权限 Token 输入框。

**生成本地哈希凭证**：

1. 在“本地哈希凭证”表单中填写：
   - **对象类型**：如 `memory`。
   - **对象 ID**：如 `mem_xxx`。
   - **凭证内容 JSON**：脱敏后的内容，如 `{"event_summary":"脱敏后的事件摘要"}`。
2. 点击“生成凭证”，系统使用 SHA-256 计算不可逆摘要并保存。
3. 凭证只保存哈希摘要，不保存隐私明文，用于证明内容在某个时间点已存在。

### 2.8 审计日志

查看所有操作记录。

**使用步骤**：

1. 在左侧导航点击 **审计日志**。
2. 页面展示所有关键操作记录，包括：
   - 记忆创建、更新、删除、清除。
   - 录音上传、转写、提炼、删除。
   - Skill 调用、工具调用。
   - 权限授予、撤销。
   - 凭证生成。
3. 每条记录包含事件类型、操作对象、时间戳和详细信息。

### 2.9 设置

管理隐私配置、模型配置和数据。

**隐私设置**：

- 是否需要 Skill 确认：控制 Skill 调用前是否需要用户确认。
- 是否允许原始记忆给工具：控制是否允许将原始记忆传递给 MCP 工具。
- 是否审计所有工具调用：控制是否记录所有工具调用的审计日志。

**模型配置**：

- Provider：模型服务提供商（如 `deepseek`）。
- 默认模型：默认调用的模型名称（如 `deepseek-chat`）。

**录音转写配置**：

- 转写 Provider：语音转写引擎（如 `whisper_local`）。

**文本提炼**：

- 在“待提炼文本”中粘贴一段文本，点击“提炼记忆卡片”，系统会自动提炼为结构化记忆卡片并写入仓库。

**数据管理**：

- **导出全部数据**：点击“导出全部数据”下载 JSON 格式的完整数据备份。
- **清除所有记忆**：点击“清除所有记忆”一键删除所有记忆卡片（此操作不可恢复，请谨慎使用）。

## 3. 隐私与安全

### 3.1 记忆脱敏说明

- 所有记忆卡片在写入前都会经过脱敏处理：真实人名、地名、公司名、账号、电话号码、精确地址和精确金额被替换为代号。
- 系统通过隐私投影层（`PrivacyProjector`）按 Skill 声明的字段对记忆卡片进行投影，仅暴露必要的脱敏字段给 Skill 和工具。
- 发送到云端模型（如 DeepSeek）的只有脱敏后的提示词和上下文，不含原始音频或原始转写全文。

### 3.2 录音本地存储说明

- 原始音频仅存储在用户本地设备的 `data/recordings/` 目录，不上传云端。
- 语音转写使用本地 `faster-whisper` 模型完成，转写过程完全离线。
- 你可以在 `config/app.json` 中设置 `auto_delete_after_days` 自动清理过期录音。
- 删除录音时会同步删除本地原始音频文件。

### 3.3 权限确认机制

- 每个 Skill 在配置文件中声明自己需要读取哪些记忆字段、是否需要用户确认。
- 当 `require_skill_confirmation` 开启时，Skill 调用前会要求用户勾选确认授权。
- 外部 Skill 和 MCP 工具不能绕过 Chat 中枢直接读取完整记忆库。
- 所有 Skill 和工具调用都会记录到审计日志。
- 你可以随时在“授权与凭证”页面撤销已授予的权限。

### 3.4 数据导出与删除

- **导出**：在“设置 → 数据管理”点击“导出全部数据”，或调用 `GET /api/v1/export`，下载 JSON 格式的完整数据。
- **删除单条记忆**：在“记忆仓库”列表中删除指定记忆。
- **删除单条录音**：在“录音仓库”列表中删除指定录音（同步删除本地音频文件）。
- **一键清除所有记忆**：在“设置 → 数据管理”点击“清除所有记忆”，或调用 `DELETE /api/v1/memories`。

> **注意**：清除所有记忆操作不可恢复，执行前请先导出数据备份。

## 4. 常见问题

### Q：API Key 如何获取？

**A**：API Key 用于保护本机接口。你可以在环境变量中设置 `SMART_AVATAR_API_KEY`，并在 `config/app.json` 中将 `security.api_key_enabled` 设置为 `true` 来启用。启用后，所有受保护的 API 请求需要在请求头中携带 `x-api-key: <你的密钥>`。模型推理使用的 DeepSeek API Key 则需要在 `.env` 文件中配置 `DEEPSEEK_API_KEY`，获取地址为 `https://platform.deepseek.com/api_keys`。

### Q：录音支持哪些格式？

**A**：系统支持以下音频格式：

- `audio/webm`（.webm）
- `audio/ogg`（.ogg）
- `audio/wav`（.wav）
- `audio/mpeg` / `audio/mp3`（.mp3）
- `audio/mp4` / `audio/m4a`（.m4a）

浏览器录音默认输出 webm 格式。你也可以通过“选择已有音频文件上传”导入上述格式的本地音频文件。

### Q：如何清除所有数据？

**A**：清除数据分为几个层级：

1. **清除所有记忆**：在“设置 → 数据管理”点击“清除所有记忆”，或调用 `DELETE /api/v1/memories`。这会删除所有记忆卡片，但保留录音和审计日志。
2. **删除单条录音**：在“录音仓库”中删除指定录音，会同步删除本地原始音频文件。
3. **导出后清除**：建议先点击“导出全部数据”备份，再执行清除操作。
4. **彻底清除**：如需彻底清除所有数据，可删除本地数据库文件 `data/smart_avatar.db` 和 `data/recordings/` 目录下的所有录音文件。

> **注意**：审计日志默认永久保留用于安全审计，不会被“清除所有记忆”操作删除。
