# 智慧分身（Memory OS）隐私政策

> 生效日期：2026-07-10
>
> 智慧分身是一个隐私优先的个人记忆与对话系统。本政策说明我们如何收集、处理、存储和保护你的数据，以及你享有的权利。

## 1. 数据收集与处理

智慧分身遵循“本地优先、最小上传”原则。原始数据默认只保留在你的设备上，云端仅接收完成脱敏、摘要和结构化后的内容。

### 1.1 记忆卡片

- 记忆卡片是脱敏后的结构化事件摘要，包含事件概要、情绪、洞察、性格侧面和实体代号等字段。
- 所有真实人名、地名、公司名、账号、电话号码、精确地址和精确金额在提炼阶段被替换为代号（例如“同事A”“常去的咖啡馆”）。
- 记忆卡片**不含原始录音**，也不含原始转写全文，只保留经过提炼和脱敏的结构化信息。

### 1.2 录音文件

- 原始音频**仅本地存储**，保存在用户设备的 `data/recordings/` 目录下，不上传云端。
- 音频在本地完成转写后，转写文本会被进一步提炼为脱敏记忆卡片。
- 原始音频文件不进入云端推理流程，也不参与模型训练。

### 1.3 用户凭证

- 用户密码使用 **bcrypt** 哈希算法存储，不可逆。系统不保存明文密码。
- 密码哈希仅存储在本地 SQLite 数据库的 `users` 表中，不出现在任何日志或导出文件中。

### 1.4 审计日志

- 系统记录所有关键操作，包括：记忆创建/更新/删除、Skill 调用、MCP 工具调用、权限授予与撤销、凭证生成等。
- 审计日志保存在本地数据库，可通过 `GET /api/v1/audit` 查看。
- 审计日志用于安全审计和操作追溯，帮助你了解“谁在什么时间对什么数据做了什么操作”。

## 2. 语音生物特征处理

语音数据属于**敏感个人信息**。系统对语音数据的处理遵循最严格的本地化策略。

### 2.1 本地存储与本地转写

- 原始音频**仅存储在用户本地设备**，不上传云端。
- 语音转写在**本地完成**，使用 `faster-whisper` 开源模型（`WhisperLocalTranscriber`）。
- 转写模型首次使用需联网下载，下载完成后完全离线运行，转写过程不依赖任何云端服务。

### 2.2 自动删除策略

- 转写完成后，原始音频可按配置自动清理。
- 自动删除周期由 `auto_delete_after_days` 配置项控制（位于 `config/app.json` 的 `transcription` 段）。
- 当 `auto_delete_after_days` 设置为具体天数时，超过该天数的录音文件将被自动删除；设置为 `null` 时表示不自动删除，需用户手动管理。

### 2.3 用户同意

- 用户首次使用录音功能前，系统会明确提示录音涉及语音生物特征等敏感个人信息，并要求用户**明确同意**。
- 用户可随时关闭录音采集，已采集的录音可随时删除。
- 涉及他人语音时，用户需自行确保录音合规并已取得相关人员的同意。

## 3. 数据存储与安全

### 3.1 本地存储

- 所有数据存储在用户**本地 SQLite 数据库**（默认路径 `data/smart_avatar.db`）。
- 记忆卡片、状态卡片、录音记录、权限授权、审计日志和凭证记录均保存在本地数据库中。
- 数据库文件不随应用镜像分发，建议对 `data/` 目录做加密磁盘或专用密钥管理。

### 3.2 接口认证

- 所有 API 端点需要 **JWT 认证**，未携带有效凭证的请求将被拒绝。
- 受保护路径以外的公共路径（如 `/`、`/static`、`/health`、`/docs`、`/openapi.json`）可匿名访问。
- 系统支持基于 `x-api-key` 的接口密钥鉴权，API Key 使用**常量时间比较**防止时序攻击。
- 系统内置基础限流（默认 120 次/分钟），并对响应添加安全头（`X-Content-Type-Options`、`X-Frame-Options`、`Referrer-Policy`、`Cache-Control`）。

### 3.3 数据导出与清除

- 支持通过 `GET /api/v1/export` 一键导出全部数据（JSON 格式）。
- 支持通过 `DELETE /api/v1/memories` 一键清除所有记忆卡片。
- 支持删除单条记忆（`DELETE /api/v1/memories/{memory_id}`）和单条录音（`DELETE /api/v1/recordings/{recording_id}`），删除录音时会同步删除本地原始音频文件。

### 3.4 凭证与证明

- 本地哈希凭证层使用 SHA-256 为记忆卡片、故事版本等内容生成不可逆摘要，用于证明内容在某个时间点已存在。
- 凭证层只保存哈希摘要，不保存隐私明文，为后续链上证明预留接口。

## 4. 用户权利

你对自己的数据享有完整的控制权。

| 权利 | 说明 | 操作方式 |
| --- | --- | --- |
| 访问权 | 查看和导出你的全部数据 | `GET /api/v1/export` 导出 JSON 格式数据 |
| 删除权 | 删除单条记忆、单条录音，或一键清除所有记忆 | `DELETE /api/v1/memories/{id}`、`DELETE /api/v1/recordings/{id}`、`DELETE /api/v1/memories` |
| 撤回权 | 随时撤销已授予 Skill 和工具的权限授权 | `POST /api/v1/permissions/{grant_id}/revoke` |
| 数据可携带权 | 以标准 JSON 格式导出数据，便于迁移 | `GET /api/v1/export` |
| 知情权 | 查看所有操作记录和数据流转 | `GET /api/v1/audit` 查看审计日志 |

## 5. 数据保留

| 数据类型 | 保留策略 |
| --- | --- |
| 记忆卡片 | **永久保留**，直到用户手动删除。记忆卡片是脱敏后的结构化摘要，是系统的核心资产。 |
| 录音文件 | 按 `auto_delete_after_days` 配置自动清理。未配置时需用户手动管理。 |
| 审计日志 | **永久保留**，用于安全审计和操作追溯。 |
| 权限授权 | 保留至用户撤销或授权过期。撤销记录仍保留在审计日志中。 |
| 用户凭证 | 密码哈希在账户存续期间保留；本地哈希凭证永久保留用于证明。 |

## 6. 第三方服务

### 6.1 DeepSeek API

- 系统使用 **DeepSeek API** 进行 Chat 对话和 Skill 的模型推理。
- 调用 DeepSeek API 时，**只发送脱敏后的提示词和上下文**，不发送原始音频、原始转写全文或未脱敏的完整记忆卡片。
- 隐私投影层（`PrivacyProjector`）会按 Skill 声明的字段对记忆卡片进行投影，仅暴露必要的脱敏字段。
- DeepSeek 不存储、不训练、不共享你的数据（具体以 DeepSeek 官方隐私政策为准）。

### 6.2 数据不出售声明

- 我们**不会将用户的原始数据出售或共享给任何第三方**。
- 第三方服务仅在最必要的情况下接收脱敏后的数据，且仅用于完成用户主动请求的功能。
- 外部 Skill 和 MCP 工具不能绕过 Chat 中枢直接读取完整记忆库，所有调用都经过权限校验和审计记录。

## 7. 联系方式

如果你对本隐私政策有任何疑问、建议或数据问题反馈，可通过以下渠道联系我们：

- **问题反馈**：在项目仓库提交 Issue，描述你遇到的数据问题或隐私顾虑。
- **数据请求**：如需访问、导出或删除你的数据，可直接通过系统内的“设置 → 数据管理”操作，或联系项目维护者。
- **隐私咨询**：如涉及语音生物特征等敏感个人信息的处理问题，请优先联系项目维护者确认处理方式。

我们承诺在收到你的数据问题反馈后尽快响应，并依法处理你的合理请求。
